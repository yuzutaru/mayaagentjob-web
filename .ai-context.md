### Geolocation Feature (Added Jul 2026)
`HeroSearchSection.tsx` displays the user's location via browser geolocation + Nominatim reverse geocode. Flow:
1. `useUserLocation` hook calls `GetUserLocationUseCase`
2. Use case delegates to `UserLocationRepositoryImpl`
3. Repository calls `navigator.geolocation.getCurrentPosition()`, then fetches Nominatim reverse geocode API (free, no key)
4. `NominatimDto.mapNominatimDtoToDomain` maps the raw API response to `UserLocation` entity
5. On failure or denial, falls back to `Jakarta, Indonesia`
6. Pure layer separation: domain has `UserLocation` + interface, data has DTO/repo, presentation has hook

## Strict Local Rules for Code Generation
1. **No Checkout Forms:** Do not code standalone payment processing gateways, transaction buttons, or checkout forms in this web repository. The web application acts purely as an evaluative/administrative view.
2. **Entitlement Enforcement:** To verify if a user has unlocked premium matching utilities, execute a standard query on the `profiles.subscription_status` data column. Entitlements are populated exclusively through mobile native execution pathways.
3. **Session Management:** Securely handle Supabase Auth token distribution across routes. Prevent layouts from rendering authenticated profile domains if the underlying session status is invalid.
4. **Autonomous Protocol:** Always output a complete Task List and Implementation Plan Artifact inside your reasoning loop before modifying component hierarchies or writing routes. Refer to [MOBILE_ONBOARDING_IMPLEMENTATION_PLAN.md](file:///Users/yuzutaru/Development/mayaagentjob-workspace/MOBILE_ONBOARDING_IMPLEMENTATION_PLAN.md) in the workspace root as a guideline.
5. **Strict Clean Architecture Layer Isolation (2026 Standard):** UI components and presentation hooks must not make raw database queries or direct ad-hoc API calls. All Supabase interactions and network requests must flow through typed repository services in the Data layer, adhering to Domain contracts for maximum stability and maintainability.
6. **Modular Architecture & Feature Isolation:** Structure frontend code into independent top-level feature modules (`src/profile/`, `src/jobs-matching/`), never nested inside an umbrella feature folder. Maintain shared infrastructure (Supabase client, base API wrappers, theme tokens) inside `src/core/` and `src/shared/`. Do not import components or repository implementations directly across feature boundaries without going through shared domain interfaces.
