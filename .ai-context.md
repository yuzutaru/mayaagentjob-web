# Local Platform Blueprint: React Web Portal (Maya)

This repository holds the desktop and web portal interface for **Maya**, built as a responsive React single-page dashboard application.

## 🛠️ Technical Stack & Architecture
* **Framework:** React 18 + Vite (Single Page Application configuration with stable modern bundling).
* **Language:** TypeScript (Strict Mode with ESLint rules enabled for 2026 best practices).
* **Architecture:** Feature-Based Modular Clean Architecture (strict vertical slicing into `presentation`, `domain`, and `data` layers).
* **Styling:** Tailwind CSS (Utility-first, responsive, dark-mode ready).
* **Database & Auth:** Supabase JS Client wrapped in repository layers.

## 🎨 Design System: Web Desktop Blue & White
The desktop layout uses a **Full-Page Scrolling Layout**:
* **HomeNavbar** - Persistent top navigation bar with brand, locale toggle, and CTA.
* **HeroSearchSection** - Search bar with location-aware greeting (browser geolocation + Nominatim reverse geocode, fallback Jakarta, Indonesia).
* **JobCategoriesBar** - Horizontal filter chips for category-based job filtering. Active category state lifted to `HomeLandingPage`; `onCategorySelect` drives `useJobListings` filter.
* **JobListingSection** - Paginated grid of job cards with AI summary bullets (9 per page). Uses `FilterJobListingsUseCase` + `MockJobListingRepository` with client-side pagination via `useJobListings` hook.
* **PopularVacanciesSection** - Highlighted vacancies with open position counts.
* **HowWeWorkSection** - Three-step workflow illustration with dotted curve connector.
* **DualCtaBannersSection** - Candidate and Employer call-to-action banners.
* **HomeFooter** - Multi-column footer with quick links.

## 🔒 Session & Entitlement Handling
* **Authentication:** Supabase Auth SDK utilizing GitHub Social Login (the primary onboarding conversion action).
* **Entitlement Model (No Direct Stripe Checkout):** This portal does not manage standalone checkout gateways. Instead, it checks the authenticated Supabase user profile table (`profiles.subscription_status`). Premium features (such as daily credits and detailed AI breakdowns) are unlocked solely if a valid subscription was initiated via the native iOS/Android mobile clients.

### Geolocation Feature
`HeroSearchSection.tsx` displays the user's location via browser geolocation + Nominatim reverse geocode. Flow:
1. `useUserLocation` hook calls `GetUserLocationUseCase`
2. Use case delegates to `UserLocationRepositoryImpl`
3. Repository calls `navigator.geolocation.getCurrentPosition()`, then fetches Nominatim reverse geocode API (free, no key)
4. `NominatimDto.mapNominatimDtoToDomain` maps the raw API response to `UserLocation` entity
5. On failure or denial, falls back to `Jakarta, Indonesia`
6. Pure layer separation: domain has `UserLocation` + interface, data has DTO/repo, presentation has hook

### Job Listing Feature
`JobListingSection.tsx` displays paginated job cards filtered by category and keyword. Flow:
1. `HomeLandingPage` creates a `MockJobListingRepository` and passes it to `useJobListings` hook
2. `useJobListings` instantiates `FilterJobListingsUseCase` internally
3. Use case delegates to `MockJobListingRepository.getFilteredListings(categoryId, keyword)` which filters `jobListingsMockData`
4. Results are paginated client-side (9 per page) via `useMemo` slice
5. `JobCategoriesBar` selection drives category filter via `handleCategorySelect` → `setCategoryId`; `HeroSearchSection` search drives keyword filter via `handleSearchSubmit` → `setKeyword`
6. `HeroSearchSection` search state (previously internal `useState`) is now lifted to `HomeLandingPage`; `onSearchQueryChange` updates input value, `onSearchSubmit` commits keyword filter
7. Both `setCategoryId` and `setKeyword` reset page to 1 on change
8. Domain entities: `JobListing.ts` (readonly interface with `jobId`, `title`, `company`, `arrangement`, `location`, `categoryId`, `matchScore`, `aiSummaryBullets`), `IJobListingRepository` (interface with `getFilteredListings`), `FilterJobListingsUseCase` (thin use case wrapping repository)
9. Mock data: 14 job listings across 6 categories (Programming & Tech, Graphics & Design, Digital Marketing, Video & Animation, Business, Writing & Translation)
10. Repository: `MockJobListingRepository` implements `IJobListingRepository`, filters by categoryId and keyword (title/company match, case-insensitive), simulates 200ms network delay

### Candidate Onboarding Feature
`useCandidateOnboardingViewModel` manages the multi-step wizard flow (`AUTH_SELECTION`, `RESUME_INGESTION`, `PROFILE_CALIBRATION`, `COMPLETED`) via `MockCandidateOnboardingRepository`. Domain contract at `src/domain/entities/CandidateOnboardingContract.ts`.

## Strict Local Rules for Code Generation
1. **No Checkout Forms:** Do not code standalone payment processing gateways, transaction buttons, or checkout forms in this web repository. The web application acts purely as an evaluative/administrative view.
2. **Entitlement Enforcement:** To verify if a user has unlocked premium matching utilities, execute a standard query on the `profiles.subscription_status` data column. Entitlements are populated exclusively through mobile native execution pathways.
3. **Session Management:** Securely handle Supabase Auth token distribution across routes. Prevent layouts from rendering authenticated profile domains if the underlying session status is invalid.
4. **Autonomous Protocol:** Always output a complete Task List and Implementation Plan Artifact inside your reasoning loop before modifying component hierarchies or writing routes.
5. **Strict Clean Architecture Layer Isolation (2026 Standard):** UI components and presentation hooks must not make raw database queries or direct ad-hoc API calls. All Supabase interactions and network requests must flow through typed repository services in the Data layer, adhering to Domain contracts for maximum stability and maintainability.
6. **Modular Architecture & Feature Isolation:** Maintain domain entities, repository interfaces, use cases in `src/domain/`, data implementations in `src/data/`, and UI/hooks in `src/presentation/`. Do not import repository implementations directly into presentation layer — use domain interfaces and dependency injection via hook parameters.
7. **Domain Contract Synchronization:** When modifying `HomePortalContract.ts`, verify corresponding files in `mayaagentjob-mobile/src/features/job-matching/domain/` remain aligned. Mobile `PopularVacancyContract` may lack `categoryId` field — ensure compatibility.
